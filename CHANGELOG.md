# Changelog - Система профилей и кодов записи

## Версия 2.0 - 12.04.2026

### 🎉 Новые возможности

#### Профили пользователей
- Добавлена команда `/profile` для просмотра профиля
- Добавлена команда `/edit_profile` для создания/редактирования профиля
- Профиль содержит: имя, телефон, дополнительную информацию
- История всех кодов записи пользователя

#### Система уникальных кодов
- Генерация 6-значных буквенно-цифровых кодов (A-Z, 0-9)
- Проверка уникальности кодов перед сохранением
- Автоматическое добавление кода в историю профиля
- Отображение кода в сообщении подтверждения записи

#### Упрощенный процесс записи
- **BREAKING CHANGE:** Убраны шаги ввода имени и телефона из процесса записи
- Данные автоматически берутся из профиля пользователя
- Обязательная проверка наличия профиля перед записью
- Новый flow: Дата → Время → Процедура → Предоплата

### 🗄️ Изменения в базе данных

#### Новые таблицы
```sql
-- Профили пользователей
CREATE TABLE user_profiles (
    user_id INTEGER PRIMARY KEY,
    name TEXT,
    phone TEXT,
    additional_info TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
)

-- История кодов записи
CREATE TABLE booking_codes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    booking_code TEXT NOT NULL,
    appointment_id INTEGER,
    created_at TEXT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES user_profiles(user_id),
    FOREIGN KEY (appointment_id) REFERENCES appointments(id)
)

-- Индекс для быстрого поиска
CREATE INDEX idx_booking_code ON booking_codes(booking_code)
```

#### Изменения в существующих таблицах
```sql
-- Добавлена колонка для хранения кода в записи
ALTER TABLE appointments ADD COLUMN booking_code TEXT
```

### ⚠️ Breaking Changes

1. **Процесс записи изменен:**
   - Удалены состояния `AWAITING_NAME` и `AWAITING_PHONE` из процесса записи
   - Теперь эти состояния используются только для редактирования профиля
   - Пользователи ДОЛЖНЫ заполнить профиль перед первой записью

2. **Новые зависимости:**
   - Процесс записи теперь зависит от наличия заполненного профиля
   - Если профиль не заполнен, запись невозможна

### 🔄 Миграция данных

При первом запуске обновленной версии:
1. Новые таблицы создаются автоматически
2. Существующие записи не затрагиваются
3. Пользователям будет предложено создать профиль при попытке записи
