# Обновление: Система профилей пользователей

## Дата: 17 апреля 2026
## Версия: 2.1.0

---

## 📋 Описание изменений

Добавлена полноценная **система профилей пользователей** с автоматическим созданием при первой записи, историей записей и управлением данными.

---

## ✨ Новый функционал

### 1. Кнопка "Мой профиль" в главном меню
- Кнопка **"👤 Мой профиль"** добавлена в главное меню для всех пользователей
- Расположена на отдельной строке для удобства доступа

### 2. Автоматическое создание профиля при первой записи
**Процесс для нового пользователя:**
1. Пользователь выбирает дату и время
2. Выбирает процедуру
3. **Система запрашивает данные для профиля:**
   - 👤 Имя (минимум 2 символа)
   - 📱 Телефон (+7XXXXXXXXXX или 8XXXXXXXXXX)
   - 📝 Комментарий (необязательно, аллергии, предпочтения)
4. Профиль создаётся автоматически
5. Процесс записи продолжается с предоплатой

**Процесс для существующего пользователя:**
- Данные берутся из профиля автоматически
- Не нужно вводить имя и телефон повторно

### 3. Просмотр профиля

**Отображаемая информация:**
```
📋 Ваш профиль

👤 Имя: Анна
📱 Телефон: +7 999 123 45 67
📝 Комментарий: Аллергия на латекс

📅 История записей:
Всего записей: 12

✅ 15.04.2026 в 14:00
   💎 Классика
✅ 10.04.2026 в 11:00
   💎 2D объём
❌ 05.04.2026 в 16:00
   💎 3D объём
...и ещё 9 записей
```

**Легенда статусов:**
- ✅ - Активная запись
- ✔️ - Завершённая запись
- ❌ - Отменённая запись

### 4. Управление профилем

**Доступные действия:**
- ✏️ **Редактировать профиль** - изменить имя, телефон, комментарий
- 🗑 **Удалить историю записей** - удалить все завершённые и отменённые записи
- ❌ **Удалить профиль** - полное удаление профиля и истории

### 5. Удаление истории записей

**Функционал:**
- Удаляются только завершённые и отменённые записи
- Активные записи сохраняются
- Требуется подтверждение перед удалением
- Показывается количество записей для удаления

**Процесс:**
```
⚠️ Удаление истории записей

Вы уверены, что хотите удалить историю записей?

⚠️ Это действие нельзя отменить!
Будет удалено записей: 8

Активные записи НЕ будут удалены.

[✅ Да, удалить историю]  [❌ Отмена]
```

### 6. Удаление профиля

**Что удаляется:**
- Имя и телефон
- Дополнительная информация
- История кодов записи
- Вся история записей (кроме активных)

**Что НЕ удаляется:**
- Активные записи остаются в системе

## 🔧 Технические изменения

### Измененные файлы

#### 1. `DatabaseService.java`
**Добавлено:**
- `getAllAppointmentsByUser(Long userId)` - получение всех записей пользователя
- `deleteAppointmentHistory(Long userId)` - удаление истории записей

```java
public static List<Appointment> getAllAppointmentsByUser(Long userId) {
    String sql = "SELECT * FROM appointments WHERE user_id = ? ORDER BY appointment_time ASC";
    return DatabaseUtils.executeQueryList(sql, rs -> mapAppointment(rs), userId);
}

public static void deleteAppointmentHistory(Long userId) {
    String sql = "DELETE FROM appointments WHERE user_id = ? AND status != 'active'";
    DatabaseUtils.executeUpdate(sql, userId);
}
```

#### 2. `ReplyKeyboardFactory.java`
**Добавлено:**
- Кнопка "👤 Мой профиль" на отдельной строке
- Расположена между основными кнопками и кнопкой отмены


**Обновлено:**

- Добавлены методы удаления истории:
  - `handleDeleteAppointmentHistoryRequest()`
  - `handleConfirmDeleteAppointmentHistory()`
  - `handleCancelDeleteAppointmentHistory()`

**Новая логика отображения:**
```java
List<Appointment> appointments = DatabaseService.getAllAppointmentsByUser(userId);
```

#### 4. `InlineKeyboardFactory.java`
**Добавлено:**
- `getProfileKeyboard()` - клавиатура с кнопкой удаления истории
- `getDeleteHistoryConfirmationKeyboard()` - подтверждение удаления истории

```java
public static InlineKeyboardMarkup getProfileKeyboard() {
    keyboard.add(List.of(makeBtn("✏️ Редактировать профиль", "edit_profile")));
    keyboard.add(List.of(makeBtn("🗑 Удалить историю записей", "delete_appointment_history")));
    keyboard.add(List.of(makeBtn("❌ Удалить профиль", "delete_profile")));
    return markup;
}
```

#### 5. `BookingHandler.java`
**Добавлено:**
- Проверка профиля при выборе процедуры
- Автоматический запрос данных для создания профиля
- Методы ввода данных:
  - `handleName()` - ввод имени
  - `handlePhone()` - ввод телефона
  - `handleAdditionalInfo()` - ввод комментария

**Новая логика:**
```java
// Проверяем профиль пользователя
Optional<UserProfile> profileOpt = DatabaseService.getUserProfile(userId);

if (profileOpt.isEmpty() || profileOpt.get().getName() == null) {
    // Профиля нет - запрашиваем данные
    BotStateManager.setState(chatId, UserState.AWAITING_NAME);
    bot.editMessageText("📋 Создание профиля...");
    return;
}

// Профиль есть - используем данные
UserProfile profile = profileOpt.get();
String name = profile.getName();
String phone = profile.getPhone();
```

#### 6. `MessageRouter.java`
**Добавлено:**
- Обработка состояний создания профиля:
  - `AWAITING_NAME` → `bookingHandler.handleName()`
  - `AWAITING_PHONE` → `bookingHandler.handlePhone()`
  - `PROFILE_EDIT_ADDITIONAL` → умная маршрутизация
- Обработка callback'ов удаления истории:
  - `delete_appointment_history`
  - `confirm_delete_history`
  - `cancel_delete_history`

**Умная маршрутизация:**
```java
stateHandlers.put(UserState.PROFILE_EDIT_ADDITIONAL, (msg, bot) -> {
    // Если есть appointmentTime - это создание профиля при записи
    if (BotStateManager.getAppointmentTime(msg.getChatId()) != null) {
        bookingHandler.handleAdditionalInfo(msg, bot);
    } else {
        // Иначе это редактирование профиля
        profileHandler.handleProfileAdditional(msg, bot);
    }
});
```

## 🎯 Преимущества

1. **Удобство для клиентов**
   - Не нужно вводить данные при каждой записи
   - История всех записей в одном месте
   - Легко отредактировать информацию

2. **Контроль данных**
   - Пользователь может удалить историю
   - Пользователь может удалить профиль
   - Прозрачность хранения данных

3. **Улучшенный UX**
   - Меньше шагов при повторной записи
   - Вся информация в одном месте
   - Понятные статусы записей

4. **Для администратора**
   - База постоянных клиентов
   - История взаимодействий
   - Контактные данные всегда актуальны

---

## 📝 Примечания

- Профиль создаётся автоматически при первой записи
- Нельзя записаться без создания профиля
- История показывает последние 5 записей
- Активные записи не удаляются при удалении истории
- При удалении профиля активные записи сохраняются в системе

---

## 🔄 Обратная совместимость

- Существующие пользователи без профиля будут запрошены создать его при следующей записи
- Старые записи остаются в базе данных
- Команды `/profile` и `/edit_profile` продолжают работать
