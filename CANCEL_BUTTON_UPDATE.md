# Обновление: Динамическая кнопка "Отменить запись"

## Дата: 17 апреля 2026
## Версия: 2.0.3

---

## 📋 Описание изменений

Добавлена **динамическая кнопка "Отменить запись"** в главное меню пользователя, которая появляется только при наличии активной записи. Отмена записи теперь требует подтверждения администратором, аналогично системе предоплаты.

---

## ✨ Новый функционал

### 1. Динамическое отображение кнопки
- Кнопка "❌ Отменить запись" появляется в главном меню **только** если у пользователя есть активная запись
- После отмены записи кнопка автоматически исчезает из меню
- После создания новой записи кнопка автоматически появляется

### 2. Процесс отмены с подтверждением админа

**Шаг 1: Клиент нажимает "Отменить запись"**
- Бот показывает детали записи и запрашивает подтверждение

**Шаг 2: Клиент подтверждает отмену**
- Запрос отправляется всем администраторам
- Клиент получает уведомление: "Запрос на отмену отправлен администратору"

**Шаг 3: Администратор принимает решение**
- ✅ **Подтвердить отмену** - запись отменяется, клиент получает уведомление
- ❌ **Отклонить** - запись остается активной, клиент получает уведомление

**Шаг 4: Обновление меню**
- При подтверждении отмены: кнопка "Отменить запись" исчезает из меню клиента
- При отклонении: кнопка остается, запись активна

## 🎯 Преимущества

1. **Удобство для клиентов**
   - Кнопка отмены всегда под рукой, если есть активная запись
   - Не нужно искать способ отменить запись

2. **Контроль для администратора**
   - Администратор может отклонить необоснованные отмены
   - Возможность связаться с клиентом перед отменой

3. **Чистый интерфейс**
   - Кнопка появляется только когда нужна
   - Меню не перегружено лишними элементами

4. **Автоматизация**
   - Меню обновляется автоматически при изменении статуса записи
   - Не требует ручного обновления

## 📝 Примечания

- Кнопка отмены добавляется **после** кнопок "Портфолио" и "Отзывы"
- Кнопка отмены добавляется **перед** кнопкой "Админ-панель" (если пользователь - админ)
- Проверка активной записи выполняется при каждом обновлении меню
- Старый метод `getMainMenuWithWebApp(boolean isAdmin)` сохранен для обратной совместимости

## 🔄 Обратная совместимость

Старый метод `getMainMenuWithWebApp(boolean isAdmin)` **сохранен** и продолжает работать без изменений. Новый метод с параметром `userId` используется только в обновленных местах.
