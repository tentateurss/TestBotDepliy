# Быстрый старт - Обновление 2.0

## Что изменилось?

### ✨ Главное изменение
Теперь пользователи **один раз** заполняют профиль (имя и телефон), а при записи выбирают только **дату, время и процедуру**. Данные подтягиваются автоматически!

## Для пользователей

### Первый раз (один раз):
```
/edit_profile
→ Введите имя
→ Введите телефон
→ (Опционально) Доп. информация
✅ Готово!
```

### Каждая запись (быстро):
```
📝 Записаться
→ Выбрать дату
→ Выбрать время
→ Выбрать процедуру
→ Оплатить и отправить скриншот
✅ Получить код записи (например: X7G3H9)
```

### Просмотр профиля:
```
/profile
→ Видите имя, телефон и историю всех кодов записи
```

## Для разработчиков

### Новые команды бота:
- `/profile` - просмотр профиля
- `/edit_profile` - создание/редактирование профиля

### Новые таблицы БД:
- `user_profiles` - профили пользователей
- `booking_codes` - история кодов записи

### Новая колонка:
- `appointments.booking_code` - код записи

### Ключевые изменения в коде:

**BookingHandler.java:**
- ❌ Удалены: `handleName()`, `handlePhone()`
- ✅ Добавлена проверка профиля в `handleBook()`
- ✅ Автоматическое получение данных из профиля в `handleServiceSelection()`
- ✅ Генерация кода в `handleConfirm()` и `handlePrepaymentConfirmation()`

**DatabaseService.java:**
- ✅ `generateUniqueBookingCode()` - генерация 6-значного кода
- ✅ `saveOrUpdateProfile()` - сохранение профиля
- ✅ `getUserProfile()` - получение профиля
- ✅ `saveBookingCode()` - сохранение кода в историю

**ProfileHandler.java (новый):**
- ✅ `handleViewProfile()` - просмотр профиля
- ✅ `handleEditProfile()` - редактирование профиля
- ✅ Обработка ввода имени, телефона, доп. информации

## Процесс записи (до и после)

### ❌ Старый процесс (7 шагов):
1. Дата
2. Время
3. Процедура
4. **Имя** ← каждый раз
5. **Телефон** ← каждый раз
6. Предоплата
7. Подтверждение

### ✅ Новый процесс (4 шага):
1. Дата
2. Время
3. Процедура
4. Предоплата
5. ✨ **Код записи**

**Имя и телефон берутся из профиля автоматически!**

## Проверка работоспособности

### Тест 1: Создание профиля
```bash
1. Запустите бота
2. Отправьте /edit_profile
3. Заполните данные
4. Проверьте /profile
```

### Тест 2: Запись без профиля
```bash
1. Новый пользователь нажимает "📝 Записаться"
2. Должно появиться сообщение о необходимости заполнить профиль
```

### Тест 3: Запись с профилем
```bash
1. Пользователь с профилем нажимает "📝 Записаться"
2. Выбирает дату, время, процедуру
3. Отправляет скриншот
4. Админ подтверждает
5. Пользователь получает код записи
6. Код появляется в /profile
```

## Миграция существующих пользователей

При первом запуске обновленной версии:
- ✅ Новые таблицы создаются автоматически
- ✅ Существующие записи не затрагиваются
- ⚠️ Пользователям нужно будет создать профиль при следующей записи

## Важные моменты

### Для администраторов:
- Сообщите пользователям о необходимости создать профиль
- Профиль создается один раз, потом только обновляется при необходимости

### Для разработчиков:
- Состояния `AWAITING_NAME` и `AWAITING_PHONE` теперь только для профиля
- Обязательно проверяйте наличие профиля перед записью
- Коды генерируются автоматически при подтверждении
