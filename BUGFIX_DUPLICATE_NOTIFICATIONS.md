# Исправление дублирования уведомлений

## Проблема

Уведомления отправлялись дважды:
```
2026-04-16T12:00:00.393+03:00  INFO 17264 --- [eduler_Worker-2] lashes.bot.scheduler.Reminder3hJob       : Sent 3h reminder for appointment 20 to user 7648345115
2026-04-16T12:00:00.418+03:00  INFO 17264 --- [eduler_Worker-1] lashes.bot.scheduler.Reminder3hJob       : Sent 3h reminder for appointment 20 to user 7648345115
```

## Причина

Уведомления планировались дважды:
1. **При создании записи** (BookingHandler.java:291-309)
2. **При подтверждении предоплаты** (BookingHandler.java:623-641)

Это приводило к тому, что в планировщике Quartz создавались два одинаковых задания для одной записи.

## Решение

Добавлена проверка перед планированием уведомлений при подтверждении предоплаты:

```java
// Планируем напоминания за 24 часа (только если ещё не запланировано)
if (appt.getReminderJobKey() == null || appt.getReminderJobKey().isEmpty()) {
    // Планируем уведомление
} else {
    log.info("24h reminder already scheduled, skipping");
}

// Планируем напоминание за 3 часа (только если ещё не запланировано)
if (appt.getReminder3hJobKey() == null || appt.getReminder3hJobKey().isEmpty()) {
    // Планируем уведомление
} else {
    log.info("3h reminder already scheduled, skipping");
}
```

## Логика работы

### При создании записи:
1. ✅ Планируется уведомление за 24 часа
2. ✅ Планируется уведомление за 3 часа
3. ✅ Сохраняются jobKey в БД
4. ❌ Уведомление админу НЕ планируется

### При подтверждении предоплаты:
1. ✅ Проверяется наличие jobKey для 24ч уведомления
   - Если есть → пропускается
   - Если нет → планируется
2. ✅ Проверяется наличие jobKey для 3ч уведомления
   - Если есть → пропускается
   - Если нет → планируется
3. ✅ Планируется уведомление админу за 2 часа
4. ✅ Планируется запрос отзыва
5. ✅ Планируется автоудаление

## Результат

Теперь каждое уведомление планируется **только один раз**:
- Уведомления за 24ч и 3ч планируются при создании записи
- При подтверждении предоплаты они пропускаются (уже запланированы)
- Уведомление админу планируется только при подтверждении
