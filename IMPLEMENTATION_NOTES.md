# Реализация профилей пользователей и системы кодов записи

## Обзор

Добавлена функциональность профилей пользователей и система генерации уникальных кодов записи для Telegram-бота.

**ВАЖНОЕ ИЗМЕНЕНИЕ:** Процесс записи упрощен - теперь пользователь указывает имя и телефон один раз в профиле, а при записи выбирает только дату, время и процедуру. Данные автоматически подтягиваются из профиля.

## Ключевые особенности

✅ **Профили пользователей** - хранение имени, телефона и дополнительной информации  
✅ **Упрощенная запись** - данные берутся из профиля, не нужно вводить каждый раз  
✅ **Уникальные коды записи** - 6-значные буквенно-цифровые коды для каждой записи  
✅ **История записей** - все коды сохраняются в профиле пользователя  
✅ **Локальное хранение** - все данные в SQLite базе данных бота

## Как это работает

### Просмотр профиля

1. Пользователь отправляет `/profile`
2. Бот показывает текущие данные профиля и историю последних 5 кодов записи
3. Если профиля нет - предлагает создать через `/edit_profile`

### Редактирование профиля

1. Пользователь отправляет `/edit_profile` или нажимает кнопку "✏️ Редактировать профиль"
2. Бот последовательно запрашивает:
   - Имя (минимум 2 символа)
   - Телефон (формат: +7XXXXXXXXXX или 8XXXXXXXXXX)
   - Дополнительную информацию (необязательно, можно пропустить через "-")
3. Профиль сохраняется в базе данных

### Система записи с кодами (ОБНОВЛЕНО)

**Новый упрощенный процесс записи:**

1. Пользователь нажимает кнопку "📝 Записаться"
2. **Проверка профиля**: Если профиль не заполнен, бот просит заполнить его через `/edit_profile`
3. Пользователь выбирает **дату** из календаря
4. Пользователь выбирает **время** из доступных слотов
5. Пользователь выбирает **процедуру** из списка
6. **Данные имени и телефона автоматически берутся из профиля**
7. Бот показывает реквизиты для предоплаты 50%
8. Пользователь отправляет скриншот оплаты
9. Админ подтверждает предоплату
10. После подтверждения:
    - Генерируется уникальный 6-значный буквенно-цифровой код (например: `X7G3H9`)
    - Код сохраняется в таблице `booking_codes`
    - Код привязывается к записи в `appointments.booking_code`
    - Код добавляется в историю профиля пользователя
    - Пользователь получает сообщение с кодом: "🎫 Ваш код записи: `X7G3H9`"

### Автоматическое создание профиля

При первой записи, если пользователь ещё не создал профиль через `/edit_profile`, бот попросит его заполнить перед началом процесса записи.

## Структура базы данных

### Таблица `user_profiles`
```sql
CREATE TABLE user_profiles (
    user_id INTEGER PRIMARY KEY,
    name TEXT,
    phone TEXT,
    additional_info TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
)
```

### Таблица `booking_codes`
```sql
CREATE TABLE booking_codes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    booking_code TEXT NOT NULL,
    appointment_id INTEGER,
    created_at TEXT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES user_profiles(user_id),
    FOREIGN KEY (appointment_id) REFERENCES appointments(id)
)
```

### Обновление таблицы `appointments`
```sql
ALTER TABLE appointments ADD COLUMN booking_code TEXT
```

## Генерация кодов

- Код состоит из 6 символов (A-Z, 0-9)
- Проверяется уникальность перед сохранением
- Максимум 100 попыток генерации (защита от бесконечного цикла)
- Пример кодов: `A3F9K2`, `X7G3H9`, `B5M8N1`

## Команды для пользователей

- `/profile` - просмотр своего профиля и истории записей
- `/edit_profile` - создание/редактирование профиля

## Особенности реализации

1. **Хранение данных**: Все данные хранятся локально в SQLite базе данных бота
2. **Безопасность**: Коды генерируются случайно и проверяются на уникальность
3. **История**: Сохраняется полная история всех кодов записи пользователя
4. **Интеграция**: Профили автоматически создаются/обновляются при записи
5. **Валидация**: Проверка формата телефона и минимальной длины имени

Для тестирования функциональности:

1. Запустите бота
2. Отправьте `/profile` - должно показать, что профиля нет
