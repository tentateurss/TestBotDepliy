# Система уведомлений - Руководство

## 📋 Обзор

Бот автоматически отправляет уведомления клиентам и администраторам о предстоящих записях. Все уведомления привязаны к часовому поясу **Самары (Europe/Samara, UTC+4)**.

## ⏰ Типы уведомлений

### 1. Уведомление клиенту за 24 часа

**Когда отправляется:** За 24 часа до записи  
**Кому:** Клиенту  
**Текст:** "⏰ Напоминаем, что завтра у вас запись на наращивание ресниц!"

**Когда планируется:**
- При создании новой записи
- При подтверждении предоплаты админом

### 2. Уведомление клиенту за 3 часа ⭐

**Когда отправляется:** За 3 часа до записи  
**Кому:** Клиенту  
**Текст:** "⏰ *{Имя}*, напоминаем, что через 3 часа у вас запись на наращивание ресниц!"

**Когда планируется:**
- При создании новой записи
- При подтверждении предоплаты админом

### 3. Уведомление админу за 2 часа ⭐

**Когда отправляется:** За 2 часа до записи  
**Кому:** Всем администраторам  
**Текст:** 
```
⏰ *НАПОМИНАНИЕ*

Через 2 часа к вам придёт клиент:
👤 {Имя}
📱 {Телефон}
💅 {Процедура}
🕐 {Время}
```

**Когда планируется:**
- **ТОЛЬКО** при подтверждении предоплаты админом

**Важно:** Это уведомление НЕ планируется при создании записи, только после подтверждения оплаты!

### 4. Запрос отзыва

**Когда отправляется:** Через 30 минут после времени записи  
**Кому:** Клиенту  
**Текст:** Просьба оставить отзыв с кнопками оценки

### 5. Автоудаление записи

**Когда выполняется:** Через 2 часа после времени записи  
**Что делает:** Автоматически помечает запись как завершённую

## 🌍 Часовой пояс

### Настройка

Часовой пояс настраивается в файле `application.properties`:

```properties
# Часовой пояс для уведомлений (Самара = Europe/Samara, UTC+4)
bot.timezone=Europe/Samara
```

### Доступные часовые пояса России:

- `Europe/Moscow` - Москва (UTC+3)
- `Europe/Samara` - Самара (UTC+4)
- `Asia/Yekaterinburg` - Екатеринбург (UTC+5)
- `Asia/Omsk` - Омск (UTC+6)
- `Asia/Krasnoyarsk` - Красноярск (UTC+7)
- `Asia/Irkutsk` - Иркутск (UTC+8)
- `Asia/Yakutsk` - Якутск (UTC+9)
- `Asia/Vladivostok` - Владивосток (UTC+10)
- `Asia/Magadan` - Магадан (UTC+11)
- `Asia/Kamchatka` - Камчатка (UTC+12)

### Как работает

1. При запуске бота в логах отображается:
   ```
   ReminderScheduler initialized with timezone: Europe/Samara
   Current time in Europe/Samara: 16.04.2026 14:30
   ```

2. Все уведомления планируются с учётом этого часового пояса

3. Время в логах отображается в формате: `dd.MM.yyyy HH:mm`

## 🛠 Техническая информация

### Файлы системы уведомлений

1. **ReminderScheduler.java** - центральный планировщик
   - Планирование всех уведомлений
   - Восстановление после перезапуска
   - Управление часовым поясом

2. **ReminderJob.java** - уведомление за 24 часа
3. **Reminder3hJob.java** - уведомление за 3 часа
4. **AdminReminderJob.java** - уведомление админу за 2 часа
5. **ReviewRequestJob.java** - запрос отзыва
6. **AppointmentCleanupJob.java** - автоудаление

### Когда планируются уведомления

**При создании записи (BookingHandler.java:302):**
- ✅ Уведомление за 24 часа
- ✅ Уведомление за 3 часа
- ❌ Уведомление админу (НЕ планируется)

**При подтверждении предоплаты (BookingHandler.java:634-645):**
- ✅ Уведомление за 24 часа (если ещё не запланировано)
- ✅ Уведомление за 3 часа (если ещё не запланировано)
- ✅ Уведомление админу за 2 часа (планируется ЗДЕСЬ!)
- ✅ Запрос отзыва
- ✅ Автоудаление

### Восстановление после перезапуска

При запуске бота автоматически восстанавливаются все уведомления для активных записей:

1. Загружаются все активные записи из БД
2. Для каждой записи проверяется, не прошло ли время уведомления
3. Если время ещё не прошло - уведомление планируется заново

---

## ⚠️ Важные моменты

### 1. Уведомление админу планируется ТОЛЬКО после подтверждения оплаты

Это сделано специально, чтобы админ получал уведомление только о подтверждённых записях.

### 2. Проверка времени

Перед планированием каждого уведомления проверяется:
```java
if (reminderTime.isBefore(now)) {
    // Пропускаем - время уже прошло
    return null;
}
```

### 3. Часовой пояс по умолчанию

Если в `application.properties` не указан часовой пояс или указан неверный, используется **Europe/Samara** по умолчанию.

### 4. Формат времени в БД

Время записи хранится в БД в формате `LocalDateTime` (без часового пояса). Часовой пояс применяется только при планировании уведомлений.

## 🧪 Тестирование

### Как проверить, что уведомления работают правильно

1. **Создайте тестовую запись на ближайшее время:**
   ```
   Например: запись на 16.04.2026 в 16:00
   Текущее время: 16.04.2026 в 14:00
   ```

2. **Проверьте логи:**
   ```
   Scheduled 3h reminder for appointment 123 at 16.04.2026 13:00 (timezone: Europe/Samara)
   Skipping 24h reminder for appointment 123 (less than 24h)
   ```

3. **Подтвердите предоплату:**
   ```
   Scheduled admin 2h reminder for appointment 123 at 16.04.2026 14:00 (timezone: Europe/Samara)
   ```

4. **Дождитесь времени уведомления** и проверьте, что сообщения отправлены

### Быстрый тест (для разработки)

Измените время уведомления на 1 минуту:
```java
// В ReminderScheduler.java
LocalDateTime reminderTime = appointment.getAppointmentTime().minusMinutes(1);
```

## 🐛 Решение проблем

**Проблема:** Уведомления приходят не в то время  
**Решение:** Проверьте настройку `bot.timezone` в `application.properties`

**Проблема:** Уведомление админу не приходит  
**Решение:** Убедитесь, что предоплата подтверждена админом (уведомление планируется только после подтверждения)

**Проблема:** После перезапуска уведомления не восстанавливаются  
**Решение:** Проверьте логи - должно быть сообщение "Restoring reminders for X active appointments"

**Проблема:** Уведомления приходят с задержкой  
**Решение:** Проверьте системное время сервера и часовой пояс ОС
